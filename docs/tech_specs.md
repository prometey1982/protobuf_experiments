# ТЗ: Клиентская часть системы запроса данных о проектах и логах

## Функциональные требования
- [ ] Запрос доступных проектов по VIN
- [ ] Запрос проекта по VIN и имени проекта
- [ ] Загрузка прошивок на сервер
- [ ] Загрузка логов на сервер

## Технические требования:
- Язык программирования: C++
- Хранилище проектов: файлы на диске
- Исходный код проекта хранится в папке src
- Передача данных: WebSockets
- Библиотека для отсылки запросов: boost.beast в асинхронном режиме
- Описание интерфейса передачи данных: уже существующий в проекте файл interface.proto
- Каждый класс должен быть описан в своем файле
- Классы однинакой семанитики, если их больше одного, следует помещать в поддиректорию
- Обращения к серверу должны идти в асинхронном режиме
- Установка соединения должна идти через TLS
- Публичный ключ, для проверки валидности, содерится в уже существующем файле python/src/srv_pub.pem
- Описание классов должны быть в .hpp файлах
- Реализация классов должна быть в .cpp файлах
- Проверка наличия проекта производится по сравнению контрольных сумм CRC32
- Если проект в наличии, то его не нужно загружать, в противополжном случае нужно запланировать асинхронную загрузку проекта
- Сгенерированные файлы для protobuf интерфейса уже находятся в папке build/proto и подключены к проекту
- Функции должны быть написаны в camel case
- Члены классов должны начинаться c _ и также в формате camel case
- Название файлов должно совпадать с названием классов
- Следует придерживаться форматирования, которое используется в уже существующих файлах
- Начальная скобка функции должна быть на новой строке
- Внутриблоковые скобки должны начинаться с той же строки
